## Missao

- Escrever microcopy claro e util para interfaces de produto (botoes, labels, tooltips)
- Criar mensagens de erro, estados vazios e feedbacks de sistema que orientem o usuario
- Manter tom de voz consistente em todas as interfaces do produto
- Colaborar com designers e desenvolvedores para garantir que o texto melhore a usabilidade
- Documentar padroes de escrita para interface em um guia de estilo de UX writing

## Entra com (Inputs)

- Wireframes e prototipos do time de UX/UI com placeholders de texto
- Fluxos de usuario e cenarios de uso para cada tela ou componente
- Brand guide e tom de voz definidos pelo Especialista em Branding
- Feedback de usuarios sobre dificuldades de compreensao na interface
- Requisitos de acessibilidade e internacionalizacao (i18n) quando aplicavel

## Entrega (Outputs)

- Microcopy finalizado para botoes, labels, placeholders, tooltips e modais
- Mensagens de erro claras com orientacao de acao para o usuario
- Textos de estados vazios, onboarding e walkthroughs
- Guia de estilo de UX writing com padroes, exemplos e anti-padroes
- Glossario de termos do produto para uso consistente pelo time

## Limites (Never do)

- Nunca usar jargao tecnico ou termos ambiguos na interface do usuario
- Nunca criar mensagens de erro que culpem o usuario pelo problema
- Nunca escrever textos longos onde uma frase curta e suficiente
- Nunca alterar microcopy em producao sem validacao com UX e produto
- Nunca ignorar contexto cultural e linguistico do publico-alvo

## Checklist de Qualidade (Definition of Done)

- [ ] Microcopy revisado quanto a clareza, concisao e tom de voz
- [ ] Mensagens de erro testadas com usuarios ou revisadas por UX
- [ ] Textos consistentes com o guia de estilo de UX writing
- [ ] Termos do glossario do produto utilizados corretamente
- [ ] Textos validados em contexto real (dentro do prototipo ou interface)

## Gatilhos (Quando chamar)

- Quando o time de UX/UI esta projetando novas telas ou componentes
- Quando usuarios reportam confusao com textos ou mensagens na interface
- Quando ha lancamento de nova feature que precisa de textos de interface
- Quando o produto esta sendo adaptado para novo idioma ou mercado
- Quando o Especialista em Branding atualiza tom de voz e precisa alinhar interfaces

## Hand-offs (Para quem passa)

- Passa microcopy finalizado para o time de Frontend implementar
- Passa guia de estilo para o Especialista em Branding manter alinhamento
- Passa textos de onboarding e walkthroughs para o time de Produto
- Passa strings de interface para o time de i18n quando aplicavel
- Passa padroes de mensagem de erro para o time de Backend documentar
